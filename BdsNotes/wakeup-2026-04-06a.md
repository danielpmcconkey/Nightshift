# Wakeup — 2026-04-06a

## What happened this session

### Project conceived

Nightshift is an autonomous software development engine. It pulls tickets
off a centralized kanban board, dispatches them through an SDLC pipeline
of AI agents, and delivers merged PRs. When it can't make a decision, it
blocks the card and moves on. Dan resolves blockers in the morning.

### Brainstorm doc written

`docs/brainstorm.md` captures the full architecture vision:

- **Board:** Postgres, no UI. One board, multiple repos via project identifier.
- **Engine:** OGRE-pattern C# state machine. Dumb orchestrator, smart agents.
- **Workflows:** DB-driven. Default SDLC pipeline + per-repo overrides.
- **Blueprints:** Standard set (proven in LeoBloom) + per-repo addenda (DSWF pattern).
- **Foreman:** New agent role for judgment calls. Narrow jurisdiction. Default is escalate.
- **Blocker protocol:** Card records where it stopped. Dan resolves in conversation with BD.
- **Morning standup:** Not CLI. A "good morning, whatcha got" conversation in Claude Code.

### Prior art identified

- **OGRE** (`/workspace/EtlReverseEngineering`): 28-node Python state machine. Task queue, agent invocation, retry logic, two-artifact-stream pattern. Port the patterns to C#.
- **Kanban board** (`/workspace/Kanban-board`): Existing C# Blazor board. Old, "much of it sucks" per Dan, but the data model might have salvageable bones. Needs assessment.
- **LeoBloom pipeline**: Battle-tested SDLC workflow (PO -> Planner -> Gherkin -> Builder -> QE -> Reviewer -> Governor -> RTE). 6+ projects shipped through it. The blueprint for the default workflow.
